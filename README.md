# Habit Tracker API

Трекер полезных привычек — серверная часть SPA-приложения, вдохновлённого книгой «Атомные привычки» Джеймса Клира.  
Сервис помогает пользователям заводить полезные привычки, получать за них вознаграждение и отслеживать прогресс.

---

## Возможности

- Регистрация и авторизация пользователей
- Создание и управление привычками
- Разделение привычек на полезные и приятные
- Привязка награды или приятной привычки к полезной
- Публичные привычки для примеров
- Telegram-бот для напоминаний
- Асинхронные задачи через Celery
- Пагинация списка привычек
- Валидация бизнес-логики привычек
- Права доступа: только владелец может управлять своими привычками

## Установка и запуск

1. Клонируйте репозиторий:

```
git clone https://github.com/Sermin22/HabitTrackerRrojectDRF
```
2. Создайте .env файл (по примеру .env_template)

3. Установите зависимости:
```
poetry install
```
4. Создайте .env файл (по примеру .env_template)
5. Примените миграции:
```
python manage.py makemigrations
python manage.py migrate
```
6. Создайте суперпользователя (по желанию):
```
python manage.py createsuperuser
```
7. Запустите сервер:
```
python manage.py runserver
```

## Архитектура

- habits — модели привычек, сериализаторы, вьюхи, валидаторы, отложенные задачи, 
сервисные функкции, тесты

- users — регистрация, авторизация (JWT), права доступа

## Валидаторы

- Только одно из полей: reward или related_habit

- Время выполнения ≤ 120 секунд

- Связанной может быть только «приятная привычка»

- У приятной привычки не может быть наград или связей

- Периодичность: не реже одного раза в 7 дней

## Права доступа

- CRUD привычек — только для авторизованных пользователей и только свои записи

- Публичные привычки доступны всем для просмотра

## Пагинация
- Стандартная пагинация по 5 привычек на страницу

## Проверка работы

После запуска сервера доступ к API осуществляется через Postman или любой HTTP-клиент.

Примеры эндпоинтов:

/habits/ — список привычек

/habits/{id}/ — детали привычки

/habits/create/ — создать привычку

/habits/{id}/update/ — изменить привычку

/habits/{id}/delete/ — удалить привычку
